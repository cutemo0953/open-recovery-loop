# The Recovery Loop (康復飛輪)

> DISCLAIMER: This is a reference framework, not medical advice. Clinical decisions must be made by qualified healthcare professionals.

## Definition

The Recovery Loop is a continuous clinical cycle for post-surgical rehabilitation:

**Evaluate → Prescribe → Exercise → Re-evaluate**

Each iteration produces data that informs the next clinical decision. The loop runs from post-operative day 1 through full recovery.

## The Four Steps

### 1. Evaluate

Assess the patient's current state using standardized measures:

- **ROM** (Range of Motion) — goniometer or digital measurement
- **VAS** (Visual Analog Scale) — daily pain score 0-10
- **Function** — sit-to-stand, walking distance, stair ability
- **PROM** (Patient-Reported Outcome Measures) — standardized questionnaires at defined timepoints
- **Wound** — photo documentation for healing progression
- **Effusion** — swelling grade (none / trace / mild / moderate / severe)

### 2. Prescribe

Create or adjust an exercise prescription based on:

- Current rehabilitation phase (determined by advancement criteria)
- Assessment findings (ROM, pain, function)
- Patient tolerance and adherence history
- Protocol template (procedure-specific, phase-specific)

A prescription includes: exercises, sets, reps, hold time, frequency, and precautions.

### 3. Exercise

The patient performs the prescribed exercises and logs completion:

- Self-directed (at home or supervised in clinic)
- Tracked per session: which exercises, actual reps, pain during exercise, difficulty rating
- Photo documentation (wound progression, swelling)
- Daily VAS reporting (independent of exercise sessions)

### 4. Re-evaluate

Reassess using the same measures as Step 1. Compare to previous evaluation:

- Is ROM progressing toward phase targets?
- Is pain trending down?
- Is adherence sufficient?
- Are there red flags requiring intervention?

Based on findings: **advance phase**, **maintain current phase**, or **regress** to previous phase.

Then prescribe again. The loop continues.

## Principles

1. **Continuous, not one-time.** The loop runs every clinic visit and between visits (via patient self-reporting).

2. **Data-driven decisions.** Each step generates structured data that informs the next step. Gut feeling is supplemented by measured trends.

3. **Override is expected.** Protocol templates are starting points. Clinician deviation (override) is clinical judgment, not failure. Override data is tracked because it is the most valuable signal for future protocol improvement.

4. **Patient is an active participant.** The "Exercise" step is performed by the patient, not done to them. Engagement and adherence are core outcomes, not just ROM and pain.

5. **Phase-based progression.** Recovery is divided into phases with explicit advancement criteria. Phases are not calendar-based — they are criteria-based.

6. **AI can suggest, clinician confirms.** AI tools may generate draft assessments or prescriptions, but no AI output directly changes clinical state without human confirmation.

## Timepoints (DSCC-aligned)

| Timepoint | When | Primary Measures |
|-----------|------|-----------------|
| Pre-op baseline | Before surgery | PROM + ROM + function |
| Post-op 6 weeks | Day 30-60 | ROM + VAS + PROM |
| Post-op 3 months | Day 75-120 | ROM + VAS + PROM + function |
| Post-op 6 months | Day 150-210 | PROM + function |
| Post-op 1 year | Day 300-425 | PROM + function |

## Who Does What

| Role | In the Loop |
|------|-------------|
| **Surgeon** | Sets initial protocol, reviews progress, confirms phase advancement |
| **PT / Therapist** | Performs assessment, adjusts prescription, guides exercise technique |
| **Patient** | Performs exercises, reports pain, completes PROMs, takes wound photos |
| **Assistant / Nurse** | Collects measurements, prepares assessment data |
| **AI (optional)** | Drafts SOAP notes, suggests prescriptions — always as draft, never auto-executed |
